================================================================
PART 1: WHERE ARE THE DIFFERENCES?
================================================================

After a line-by-line review of all 6 files, the Gemini-web versions
are consistently better in 8 specific categories:


---------------------------------------------------------------
1. NARRATIVE CHOREOGRAPHY (async/await staged storytelling)
---------------------------------------------------------------
ORIGINAL: Uses raw setTimeout chains and simple onclick handlers.
          Example (Sec1): onclick="startSequence()" → nested setTimeout
          callbacks with hardcoded delays. No narrator text updates.

GEMINI:   Uses async/await with a sleep() helper and a "narrator"
          text element that updates at each stage of the animation.
          Example: narrator.textContent = "Time Evolution: Digital
          clock tracking across instructional days."
          Each stage is clearly separated: STAGE 1, STAGE 2, etc.

IMPACT:   The Gemini approach creates a GUIDED EXPERIENCE where the
          user reads what's happening. The original is silent — you
          just watch things move with no context.

APPLY TO: ALL 37 scenes (Lec4-Lec12). Every scene needs:
          - A narrator <p> element at the top
          - async/await flow replacing setTimeout chains
          - Stage labels in the JS code


---------------------------------------------------------------
2. FULL RESET & REPLAY CAPABILITY
---------------------------------------------------------------
ORIGINAL: No reset logic. After the animation plays once, the button
          is either disabled or does nothing meaningful. Refreshing
          the page is the only way to replay.

GEMINI:   Every scene has a dedicated resetScene() or equivalent
          function that restores ALL element states (classes, styles,
          positions, opacity) to their initial values. The button
          text changes to "Replay Simulation" after completion.

IMPACT:   Users can replay the visualization without refreshing.
          Critical for educational use where students review content.

APPLY TO: ALL 37 scenes. Every scene needs:
          - A reset function that undoes every visual change
          - Button text update: "Start" → "Replay"
          - Guard variable (isRunning) to prevent double-clicks


---------------------------------------------------------------
3. TRAVELING SIGNAL / MOVING INDICATOR
---------------------------------------------------------------
ORIGINAL (Sec1): The timeline axis fills with color via a single
          CSS width transition. Nodes activate but there's no visible
          "cursor" or "signal" traveling along the timeline.

GEMINI (Sec1):   A glowing white dot (time-signal) physically moves
          from left to right along the timeline bar, stopping at each
          node. This creates a clear sense of time progression.

IMPACT:   The traveling signal gives the user a focal point to follow.
          Without it, the eye doesn't know where to look.

APPLY TO: Applicable scenes that have timelines, iteration paths,
          or sequential flows (Lec5/Sec4, Lec9/Sec1, Lec9/Sec3,
          Lec9/Sec4, all of Lec10). Add a visible traveling dot/pulse
          that leads the eye through the animation sequence.


---------------------------------------------------------------
4. DIFFERENTIATED NODE SHAPES & VISUAL ENCODING
---------------------------------------------------------------
ORIGINAL (Sec2): All data clusters are circles (border-radius: 50%).
          The only differentiator is color. Data points inside are
          all identical white dots.

GEMINI (Sec2):   Three distinct shapes per data type:
          - Panel Data = rounded-sm (squares)
          - Categorical = shape-triangle (CSS clip-path)
          - Longitudinal = rounded-full (circles)
          Additionally, 54 small particle nodes (18 per cluster) are
          generated dynamically with dataset attributes (relevant,
          type) for individual filtering.

IMPACT:   Shape encoding follows data visualization best practices.
          Users can distinguish types even in grayscale. Individual
          particles allow granular filtering animations.

APPLY TO: Lec7/Sec1 (Normal vs Exponential), Lec7/Sec2 (Poisson
          spikes), Lec10/Sec5 (three test statistics). Any scene with
          multiple data categories should use distinct shapes, not
          just color.


---------------------------------------------------------------
5. PENALIZATION / SWEEP WAVE EFFECT
---------------------------------------------------------------
ORIGINAL (Sec2): The penalization wave is a rotated div that sweeps
          via CSS @keyframes once. After sweep, nodes randomly get
          filtered based on slider value — no deterministic logic.

GEMINI (Sec2):   The wave is a blurred gradient (cyan + rose) that
          sweeps left-to-right via CSS transition (not keyframes).
          When the wave passes the midpoint (~1 second), a setTimeout
          fires that deterministically classifies each node:
          - relevant=true → scale(1.3) + box-shadow glow
          - relevant=false → opacity 0.15, scale(0.4), grayscale
          The wave then fades out (opacity: 0), leaving the result.

IMPACT:   The Gemini approach feels "intelligent" — like a scanner
          that reveals hidden structure. The original feels random.

APPLY TO: Lec8/Sec3 (Canonical cancellation), Lec11/Sec4 (Pearson
          vs Deviance), any scene where a filter or selection process
          separates elements. Use a visible sweep effect followed by
          deterministic classification.


---------------------------------------------------------------
6. FINAL STATE MAPPING WITH MODEL TARGET BOXES
---------------------------------------------------------------
ORIGINAL (Sec2): After penalization, nodes remain in the workspace
          with modified styling. No "final destination" is shown.
          There are no labeled model boxes.

GEMINI (Sec2):   After penalization, three labeled model boxes fade in
          (Panel → Fixed/Random Effects, Categorical → GLM/Logistic,
          Longitudinal → Mixed-Effects). Nodes then physically move
          into their respective boxes. Relevant nodes sit centered;
          redundant nodes drop lower. A "Misconception Corrected"
          overlay with rose styling appears.

IMPACT:   The final state provides clear takeaway learning. Users see
          the concrete mapping: Data Type → Correct Model. Without
          this, the visualization is abstract without conclusion.

APPLY TO: Lec6/Sec1 (unified framework), Lec7/Sec3 (grouped data),
          Lec10/Sec1 (hypothesis formation). Every scene should have
          a clear "final state" panel or message summarizing what was
          demonstrated, not just fading out.


---------------------------------------------------------------
7. ANIMATED METERS WITH LIVE VALUE DISPLAYS
---------------------------------------------------------------
ORIGINAL (Sec2): Sliders exist but are manual (user-controlled).
          No animated meters. No value labels updating in real-time.

GEMINI (Sec2):   Three meters with labels and live value readouts:
          - "Structural Filter" showing 0% → 100%
          - "L1/L2 Penalization" showing 0.0 → 10.0
          - "Noise Reduction" showing Off → Engaging → Optimized
          All animate automatically via an animateMeters() async loop.
          Meters are set to pointer-events-none (non-interactive)
          during simulation.

IMPACT:   Animated meters give a sense of "processing" and scientific
          rigor. The user feels like a machine is working, not just
          playing a movie.

APPLY TO: Lec5/Sec4 (Newton-Raphson convergence), Lec8/Sec4 (Fisher
          Information), Lec8/Sec5 (Dispersion estimation), Lec9/Sec1
          (convergence threshold). Any scene with iterative processes
          should show meters or gauges that animate during processing.


---------------------------------------------------------------
8. SCATTERED INITIAL STATE → ORGANIZED FINAL STATE
---------------------------------------------------------------
ORIGINAL (Sec3): Blocks start offscreen (translateY: -800px) and
          drop into position via CSS transition-delay staggering.
          This is a simple "drop from above" pattern.

GEMINI (Sec3):   Blocks start scattered randomly in space (random X,
          random Y, random rotation 0-30°) and then fly into their
          exact stacked positions. The anchor block ("Basic Concepts")
          is already in place as the foundation. A GLOWING PULSE
          element moves upward through the stack, highlighting each
          block in sequence, revealing "Slides" and "Hand-written"
          overlays inside each block.

IMPACT:   The scattered → organized pattern creates a "chaos to
          order" narrative. The pulse scanning upward reinforces the
          idea of logical continuity between levels.

APPLY TO: Lec6/Sec2 (exponential family), Lec7/Sec3 (individual
          → grouped), Lec11/Sec2 (deviance accumulation). Any scene
          where elements need to "come together" should use random
          scattered initial positions, not just offscreen drops.

PART 2: THE UPDATE PLAN

PRIORITY: HIGH (Structural changes)

PHASE A — UNIVERSAL INFRASTRUCTURE (Apply to ALL 37 scenes)
---------------------------------------------------------------
A1. Add narrator element
    - Insert <p id="narrator"> below every <h1> title
    - Style: color: #94a3b8, font-size: 14px, max-width: 800px
    - Fixed height (h-8 or height: 32px) to prevent layout shift

        const sleep = ms => new Promise(r => setTimeout(r, ms));

A3. Add reset + replay logic
    - Create resetScene() function per file
    - Restore ALL modified styles/classes to initial state
    - Button text: "Start" → "Simulating..." → "Replay"
    - Clear any dynamically created elements (SVG lines, particles)

A4. Add "Final State" summary panel
    - Every scene should end with a visible takeaway message
    - Style: bg-[color]-950/80 border border-[color]-500/30
      backdrop-blur rounded-xl p-5 text-center
    - Appears with opacity + translateY transition
